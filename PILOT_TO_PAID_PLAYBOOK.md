# Pilot-to-Paid Conversion Playbook

## Entry Criteria
1. Pilot completed with signed success criteria.
2. Benchmark evidence and release evidence available.
3. No unresolved critical defects for target use case.

## Conversion Steps
1. Technical close:
   1. Confirm measured deltas vs baseline.
   2. Confirm integration/support requirements.
2. Commercial close:
   1. Choose package tier.
   2. Confirm term, scope, and support SLA.
3. Operational close:
   1. Define production rollout checkpoint dates.
   2. Define escalation and vulnerability response channels.

## Exit Conditions
1. `convert`: procurement/legal complete.
2. `extend`: time-boxed extension with explicit remaining gap.
3. `stop`: no path to production value within agreed window.
